Home Page

The Home Page acts as the central entry point of the website, introducing the theme "Life on Land" and providing users with an overview of key environmental topics. The structure is created using semantic HTML tags such as <header>, <main>, and <section> to organize content clearly and improve accessibility.

A hero section is created using a <section id="home"> element, which contains a large title displayed using the <h1> tag ("Land Verse") and a tagline using a <p> tag. This creates a strong visual hierarchy and first impression.

The "Mission & Goals" section is structured using <div class="cards-container">, which contains multiple <div class="card"> elements. Each card includes:
• <img src=""> for images
• <h3> for headings
• <p> for descriptions
• <ul> and <li> for action points

One of the cards is wrapped inside an <a href=""> tag, making the entire card clickable and linking it to another content page. This improves navigation and encourages user interaction.

The layout is designed using CSS Flexbox, with properties such as display: flex;, justify-content, and align-items to align the cards properly and ensure responsiveness across different screen sizes.

The navigation bar is built using <nav>, <ul>, and <li> elements, with links created using <a href="">. The active page is highlighted using a class such as class="active".

The footer is created using the <footer> tag and contains multiple <div> sections for contact details, editors, and page links, ensuring a consistent structure across the website.

Gallery Page

The Gallery Page is designed to visually represent important environmental topics through images and short descriptions. The structure is created using <section>, <div>, and <p> tags to organize content clearly and improve readability.

The page displays multiple image cards using <div class="img-box"> containers. Each card includes a title using <p>, an image using the <img src=""> tag, and a description using another <p>. These topics include forest conservation, wildlife protection, reforestation, and land degradation.

A key feature of this page is the interactive modal system implemented using JavaScript. The modal is created using a hidden <div id="modal"> element. Inside it, elements like <img>, <h2>, and <p> are used to display the selected image, title, and description dynamically.

When a user clicks on an image, JavaScript event listeners are triggered, updating the modal content by changing attributes such as src of the <img> tag and text content inside <h2> and <p>.

This functionality improves user experience by allowing users to focus on specific content without navigating away from the page. It also makes the website more dynamic and interactive.

The layout is structured using CSS, particularly Flexbox with display: flex;, to arrange image cards neatly. Spacing is controlled using properties like margin and gap, ensuring consistent alignment across different screen sizes.

Additional styling such as :hover effects and transform properties are used to enhance interactivity, making images more engaging when users interact with them.

Content Page – Protect Forests

The Content Page titled "Protect Forests" focuses on raising awareness about the importance of forests and the need for sustainable environmental practices. The page structure is built using semantic HTML tags such as <main>, <section>, and <h1> to organize content clearly and improve accessibility.

The layout is divided into multiple sections using <section id=""> elements, each addressing a specific aspect of forest conservation. These sections include headings created with <h2> and paragraphs using <p> to present information in a structured format.

Each section follows a consistent design using a split layout created with <div class="text"> and <div class="image">. Images are displayed using the <img src=""> tag, while text content is placed in paragraphs. This side-by-side layout is controlled using CSS Flexbox (display: flex;) to improve readability.

The alternating layout is achieved using a class such as class="content-section reverse", where the CSS property flex-direction: row-reverse; switches the position of text and images. This prevents the page from looking repetitive and improves visual design.

An internal navigation bar is created using a <div class="internal-nav"> containing multiple <a href="#section-id"> links. These links allow users to quickly jump to sections like #importance or #climate, improving navigation within the page.

The page also incorporates interactive design elements such as hover effects. For example, images styled with .content-img use transform: scale() on :hover to create a zoom effect, while content sections use transform: translateY() and box-shadow to create a lifting animation.

The design is built using HTML for structure and CSS for styling. Layout is controlled using Flexbox properties such as display: flex;, gap, and justify-content. Additional styling like background: rgba() and backdrop-filter: blur() creates a modern glass effect.

Navigation between pages is implemented using the <a href="contentProtectForests.html"> tag, allowing users to move between different parts of the website easily.

Overall, the Content Page combines structured HTML elements with CSS styling and interactive effects to create an informative, visually appealing, and user-friendly experience.

Technical Description

The website was developed using HTML for structure, CSS for styling, and JavaScript for interactivity. For example, structural elements such as <header>, <main>, <section>, and <footer> are used to organize the layout. External CSS files like global.css, home.css, and gallery.css are linked using the <link> tag to maintain consistency and improve code organization.

Flexbox was used for layout design using properties such as display: flex;, justify-content, and align-items. For example, navigation menus are structured using <nav>, <ul>, and <li> tags, and styled with Flexbox to align items properly. This allows the layout to adjust responsively across different screen sizes.

The Gallery Page uses JavaScript to handle user interactions. Images are created using the <img> tag, and when clicked, JavaScript event listeners update elements inside a modal such as <div id="modal">, <img>, <h2>, and <p>. This allows dynamic content changes without reloading the page.

The navigation system is consistent across all pages using <a href=""> links. The active page is highlighted using a class such as class="active". Reusable components like headers and footers are implemented using <header> and <footer> tags, ensuring a uniform design throughout the website.

Challenges and Lessons Learned

One of the main challenges was ensuring consistency across multiple pages while using different CSS files. Maintaining the same design style required careful planning and reuse of common classes.

Another challenge was implementing the modal functionality for the gallery. It required understanding how JavaScript interacts with HTML elements and how to dynamically update content.

I also faced difficulties with spacing, alignment, and responsiveness. These were solved by using Flexbox and testing the layout on different screen sizes.

Through this project, I improved my skills in front-end development, especially in structuring web pages, managing styles, and creating interactive features.

Compliance

This website was developed in accordance with the given coursework guidelines and requirements. All pages follow a consistent structure, including a navigation bar, main content area, and footer, ensuring a uniform user experience across the site.

The design maintains accessibility by using clear fonts, readable text sizes, and sufficient color contrast between text and background. Navigation is simple and intuitive, allowing users to easily move between pages using the menu and internal links.

The website is responsive and designed using modern CSS techniques such as Flexbox, ensuring that content is displayed properly on different screen sizes and devices.

External files such as CSS and JavaScript have been properly linked, and the code is organized and structured to improve readability and maintainability. Reusable components like headers and footers are consistently used throughout the website.

Overall, the website meets the required web development standards and follows best practices in design, usability, and content organization.
