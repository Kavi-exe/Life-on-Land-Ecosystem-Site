Feedback Page

The Feedback Page allows users to share their thoughts and experiences with the "Life on Land" initiative. The structure is built using semantic HTML elements including <section>, <form>, <fieldset>, and <legend> to ensure accessibility and clear organization.

I implemented a dynamic program directory using <div class="program-container"> containing multiple <div class="card"> elements. Each card includes an image using <img> tag, title with <h3>, description with <p>, and an interactive button. A custom dropdown <select> allows users to filter programs by categories such as "Most Popular", "Nearest", and "Student-Friendly".

The feedback form collects personal details (name, email, telephone) and feedback-specific fields including program selection and message. I integrated real-time character counting using JavaScript: an event listener on the <textarea id="message"> updates the <small id="counter"> element dynamically, displaying remaining characters with a maximum limit of 150 characters.

Client-side validation was implemented to ensure name and email fields are not empty. Error messages appear inside <div class="error"> elements. Upon successful submission, a success message is displayed and the form resets automatically. The reviews section showcases user ratings using structured <div class="review-card"> containers, providing social proof and encouraging user engagement.

The design follows a dark theme with glassmorphism effects using background: rgba() and backdrop-filter: blur(), consistent with the global aesthetic. Hover effects on cards and form elements enhance interactivity. External CSS (feedback.css) manages layout with Flexbox, while JavaScript (feedback.js) handles all interactive functionality.

Team Page

The Team Page introduces the four members behind Land Verse, highlighting their roles and individual contributions to the project. The layout uses a Flex-based grid with .team-container to display responsive member cards.

Each <div class="team-card"> contains a profile image using <img> tag, name with <h2>, role with <p>, and an expandable details section that reveals contributions on hover or focus. This expandable feature is implemented using CSS transitions with max-height animation, creating a smooth user experience.

Accessibility was prioritized by including tabindex="0" on each card, making the hover effect keyboard-navigable for users who rely on keyboard navigation. The contribution text provides specific details about each student's assigned tasks, ensuring transparency about project responsibilities.

All images include meaningful alt attributes describing each team member for screen reader compatibility. The overall design follows the global typography with font-family: 'f1' and color scheme, featuring a dark background with subtle box-shadow effects on hover to indicate interactivity.

Content Page – Safeguarding Wildlife

The Content Page titled "Safeguarding Wildlife" focuses on raising awareness about the importance of wildlife conservation and the need for sustainable environmental practices. The page structure is built using semantic HTML tags such as <main>, <section>, and <h1> to organize content clearly and improve accessibility.

The layout is divided into multiple sections using <section id=""> elements, each addressing a specific aspect of wildlife conservation. These sections include headings created with <h2> and paragraphs using <p> to present information in a structured format. Topics covered include importance of wildlife, biodiversity and ecosystem balance, major threats to wildlife, impact of climate change, wildlife protection methods, laws and conservation efforts, and how individuals can help.

Each section follows a consistent design using a split layout created with <div class="text"> and <div class="image">. Images are displayed using the <img src=""> tag, while text content is placed in paragraphs. This side-by-side layout is controlled using CSS Flexbox (display: flex;) to improve readability and visual appeal.

The alternating layout is achieved using a class such as class="content-section reverse", where the CSS property flex-direction: row-reverse; switches the position of text and images. This prevents the page from looking repetitive and improves visual design by alternating content positioning.

An internal navigation bar is created using a <div class="internal-nav"> containing multiple <a href="#section-id"> links. These links allow users to quickly jump to sections like #importance, #biodiversity, #threats, #climate, #protection, #laws, and #how-to-help, improving navigation within the page.

The page incorporates interactive design elements such as hover effects. For example, images styled with .content-img use transform: scale() on :hover to create a zoom effect, while content sections use transform: translateY() and box-shadow to create a lifting animation when hovered, enhancing user engagement.

The design is built using HTML for structure and CSS for styling. Layout is controlled using Flexbox properties such as display: flex;, gap, and justify-content. Additional styling like background: rgba() and backdrop-filter: blur() creates a modern glass effect that makes the content visually appealing while maintaining readability.

Technical Description

The website was developed using HTML5 for semantic structure, CSS3 for styling and responsive design, and vanilla JavaScript for interactivity. Structural elements such as <header>, <main>, <section>, and <footer> are used consistently across all pages to organize content logically.

External CSS files including global.css, feedback.css, team.css, and content.css are linked using the <link> tag to maintain modularity and improve code organization. This approach allows for easier maintenance and consistency across the website.

For the Feedback Page, I implemented form validation without external libraries. JavaScript event listeners handle form submission, check for empty required fields, and display error messages dynamically. The character counter uses the input event to update remaining characters in real-time.

The Team Page utilizes CSS transitions and hover effects to reveal hidden details, enhancing user interface without complex JavaScript. The expandable card feature is achieved using max-height transitions, providing a smooth animation when users interact with the cards.

The Content Page demonstrates advanced CSS techniques including Flexbox layouts, alternating content sections using flex-direction: row-reverse, and glassmorphism effects with backdrop-filter: blur(). Internal anchor navigation provides seamless scrolling between sections, improving user experience for longer content pages.

The navigation system is consistent across all pages using <a href=""> links, with active pages highlighted using class="active". The "Top ↑" button implemented with JavaScript provides convenient navigation back to the top of long pages.

Challenges and Lessons Learned

One of the main challenges was implementing the alternating layout for the Content Page sections while ensuring responsiveness across different screen sizes. Using flex-direction: row-reverse with media queries required careful testing to maintain proper layout on mobile devices. This taught me the importance of planning responsive designs from the start.

Another challenge was implementing the expandable details on the Team Page to be accessible for keyboard users. Adding tabindex="0" and ensuring CSS transitions work properly on both hover and focus states required careful testing and refinement.

I also faced difficulties with form validation edge cases, such as handling empty submissions and resetting the character counter correctly after form submission. Thorough testing and debugging helped resolve these issues, improving my understanding of JavaScript event handling and DOM manipulation.

Creating the internal navigation system for the Content Page with smooth scrolling and active section highlighting was a valuable learning experience in implementing anchor-based navigation and scroll event handling.

Through this project, I significantly improved my front-end development skills, particularly in creating interactive features, managing responsive layouts with Flexbox, implementing form validation, and ensuring accessibility standards across all pages.

Compliance

The design maintains accessibility standards by using clear fonts, readable text sizes, and sufficient color contrast between text and background. All images include descriptive alt attributes for screen reader compatibility, and semantic HTML elements provide proper document structure.

All images used in the website are sourced from free and copyright-free platforms such as Unsplash and Pexels, with proper references included. No copyrighted material has been used without permission.

The website is fully responsive and designed using modern CSS techniques including Flexbox, CSS Grid, and media queries, ensuring that content is displayed properly on different screen sizes from mobile devices to desktop monitors.

External files such as CSS and JavaScript have been properly linked, and the code is organized and structured to improve readability and maintainability. Reusable components like headers and footers are consistently used throughout the website.

Overall, the website meets the required web development standards and follows best practices in design, usability, accessibility, and content organization.
