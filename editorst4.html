Sitemap Page

The Sitemap Page provides a visual overview of the website's structure and hierarchy, helping users easily navigate the "Life on Land" project. The structure is built using semantic HTML elements such as <header>, <main>, and <section> to organize content clearly.

A standout feature is the interactive and responsive visual map, implemented using the <svg viewBox="..."> element to define scalable vector graphics.

Inside the SVG, navigation paths between the nodes are drawn using <path class="link-line" d="..."> elements, creating a clear tree structure that outlines the connections between parent pages and their subpages.

The interactive clickable nodes in the sitemap are constructed using SVG elements: <rect> for the button shapes and <text> for the labels. These elements are wrapped inside <a href="..." class="node-group"> tags, ensuring that clicking on any node safely redirects the user to the respective page like Home, Gallery, Feedback, or Validation.

To ensure inclusivity and adherence to accessibility standards, the SVG container and the link tags utilize role="navigation" and aria-label="..." attributes, providing clear context to screen readers.

User Profile Page

The User Profile Page allows visitors to create a personalized profile indicating their engagement and interests related to "Life on Land" conservation. Its structure relies on HTML and heavily utilizes JavaScript for collecting and displaying dynamic data.

The layout implements a progressive reveal design. A hidden progress tracker is created using an inline style="display: none;" on the <div id="progress-wrapper"> element. It encloses a <div class="progress-bar"> to visually indicate the profile completion status based on user input.

Instead of a traditional form, data collection relies on JavaScript prompts to achieve an interactive step-by-step experience. Action buttons like 'Start Profile Building' and 'Reset Profile' are mapped to JavaScript logic via the onclick attribute (e.g., onclick="startProfileBuilding()").

When logic executes in JavaScript, responses are collected and DOM manipulation occurs immediately. The script maps answers via document elements like document.getElementById("out-name").textContent, allowing the profile summary (built with headings and spans) to reflect inputs in real time.

Advanced logic tracks the percentage of completion. By storing prompt responses in arrays and checking variable states, the page allows users to skip questions, updates the progress bar visually, temporarily hides answered fields, permits resuming skipped steps securely via the resumeSkippedSteps() function, and can wipe all data clear dynamically using the resetProfile() function.

Content Page – Restoring Soil Health

The Content Page titled "Restoring Soil Health" focuses on raising awareness about the importance of maintaining healthy soil and promoting sustainable land management practices. The page is structured using semantic HTML elements such as <main>, <section>, and <h1>.

The layout is divided into multiple sections using <section id=""> elements, covering topics such as soil importance, biodiversity, threats, climate impact, restoration methods, and solutions.

Each section uses a split layout with <div class="text"> and <div class="image">. This improves readability and keeps the design visually balanced.

Alternating layouts are created using class="content-section reverse", which applies flex-direction: row-reverse; in CSS to avoid repetition.

Internal navigation is implemented using <div class="internal-nav"> with anchor links for easy section access.

Interactive effects such as hover zoom on images (transform: scale()) and card lift animations (transform: translateY()) enhance user experience.

The page is styled using external CSS files (global.css, content.css) and uses Flexbox for responsive layout design.

Navigation to this page is implemented using <a href="content_ST4.html">.

Overall, the page combines structured HTML, CSS styling, and interactive elements to create an engaging and informative user experience.

Technical Description

The website was developed using HTML for structure, CSS for styling, and JavaScript for interactivity. For example, structural elements such as <header>, <main>, <section>, and <footer> are used to organize the layout. External CSS files like global.css, sitemap.css, and user_profile.css are linked using the <link> tag to maintain consistency and improve code organization.

Flexbox was used for layout design using properties such as display: flex;, justify-content, and align-items. For example, navigation menus are structured using <nav>, <ul>, and <li> tags, and styled with Flexbox to align items properly. This allows the layout to adjust responsively across different screen sizes.

The Sitemap Page uses SVG graphics specifically styled to maintain aspect ratios using a coordinate system viewBox="0 0 1000 600" allowing vectors to be naturally scalable. The User Profile Page uses fundamental Vanilla JavaScript methods to orchestrate user flows, handle display states dynamically between block or none, calculate percentage formulations mathematically, and update inner DOM elements without requesting server refreshes.

The navigation system is consistent across all pages using <a href=""> links. The active page is highlighted using a class such as class="active". Reusable components like headers and footers are implemented using <header> and <footer> tags, ensuring a uniform design throughout the website.

Challenges and Lessons Learned

One of the main challenges was drawing and calculating precise paths between nodes in the SVG coordinate system for the Sitemap. It took multiple iterations to align visually pleasing <path> markers with <rect> node origins correctly.

Another significant hurdle was structuring the modular logic inside the User Profile. Building a progressive step tracker relying sequentially on prompt loops meant carefully managing scope and correctly addressing skipped inputs rather than crashing or skipping the validation checks completely.

I also faced difficulties with making SVG responsive and text easily readable across screens, solved by removing absolute sizing and relying strictly on viewBox parameters.

Through this project iteration, I learned considerable detail about Scalable Vector Graphics formatting within HTML and expanded my JavaScript DOM interaction abilities remarkably.
